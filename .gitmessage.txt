# 🎯 커밋 유형 선택 (한 줄만 선택 후 아래 설명 작성)
#
# feat:     새로운 기능 추가
# fix:      버그 수정
# refactor: 리팩터링 (기능 변화 없음)
# style:    코드 스타일 변경 (공백, 포맷 등)
# docs:     문서 수정 (README, 주석 등)
# test:     테스트 추가 또는 수정
# chore:    기타 변경사항 (빌드, 설정 등)
# build:    빌드 시스템 관련 변경
# ci:       CI 관련 설정 변경
# revert:   이전 커밋 되돌리기

# ✍️ 예시:
# feat: 기록 생성 기능 추가
# fix: 로그인 시 토큰 저장 오류 수정

<type>: <변경 내용 요약>
